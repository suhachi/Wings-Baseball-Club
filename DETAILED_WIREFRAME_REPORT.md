### 1.1 앱 전체 구조도

```
┌─────────────────────────────────────────┐
│           TopBar (고정)                  │
│  [←] [로고+제목] [🔔알림] [⚙️설정]      │
├─────────────────────────────────────────┤
│                                         │
│         Main Content Area               │
│         (페이지별 컨텐츠)                │

│         (스크롤 가능 영역)               │
```

### 1.2 레이아웃 상세 사양

- **최대 너비**: `max-w-md` (약 448px, 모바일 최적화)
- **배경색**: `bg-gray-50 dark:bg-gray-950`
- **상단 바 높이**: `h-14` (56px) + `pt-safe` (Safe Area)
- **하단 네비게이션 높이**: `h-16` (64px) + `pb-safe` (Safe Area)
- **메인 컨텐츠 패딩**: `pb-20 pt-16` (하단 네비게이션 공간 확보)

## 2. 공통 컴포넌트

### 2.1 TopBar (상단 바)

**위치**: `src/app/components/TopBar.tsx`

#### 레이아웃 구조

```
┌─────────────────────────────────────────┐
│ [←]  [로고 이미지] 제목  [🔔] [⚙️]    │
└─────────────────────────────────────────┘
```

#### 구성 요소

| 위치 | 요소 | 조건부 표시 | 기능 |
|------|------|------------|------|
| 좌측 | 뒤로가기 버튼 (`ArrowLeft`) | `showBack === true` | 이전 페이지로 이동 |
| 중앙 | 로고 이미지 (`wingslogo.jpg`, 32x32, 둥근 모서리) | 항상 | 클릭 시 홈으로 이동 (선택적) |
| 중앙 | 페이지 제목 (텍스트) | 항상 | 현재 페이지 제목 표시 |
| 우측 | 알림 아이콘 (`Bell`) | `showNotification === true` | 알림 페이지로 이동, 미읽은 알림 뱃지 표시 |
| 우측 | 설정 아이콘 (`Settings`) | `showSettings === true && isAdmin()` | 설정 페이지로 이동 (관리자만) |

#### 스타일

- **배경**: 그라데이션 `from-blue-600 to-blue-500` (다크모드: `from-blue-700 to-blue-600`)
- **높이**: `h-14` (56px)
- **고정 위치**: `fixed top-0 left-0 right-0 z-40`
- **Safe Area**: `pt-safe` (iOS 노치 대응)

#### Props

```typescript
interface TopBarProps {
  title?: string;                    // 페이지 제목
  showBack?: boolean;                // 뒤로가기 버튼 표시 여부
  onBack?: () => void;               // 뒤로가기 클릭 핸들러
  showNotification?: boolean;        // 알림 아이콘 표시 여부
  showSettings?: boolean;            // 설정 아이콘 표시 여부
  onNotificationClick?: () => void;  // 알림 클릭 핸들러
  onLogoClick?: () => void;          // 로고 클릭 핸들러
  unreadNotificationCount?: number;  // 미읽은 알림 개수
}
```

### 2.2 BottomNav (하단 네비게이션)

**위치**: `src/app/components/BottomNav.tsx`

#### 레이아웃 구조

```
┌─────────────────────────────────────────┐
│  [홈]  [일정]  [게시판]  [앨범]  [내정보] │
└─────────────────────────────────────────┘
```

#### 탭 구성

| 탭 ID | 아이콘 | 라벨 | 대상 페이지 |
|-------|--------|------|------------|
| `home` | `Home` | 홈 | HomePage |
| `schedule` | `Calendar` | 일정 | SchedulePage |
| `boards` | `MessageSquare` | 게시판 | BoardsPage |
| `album` | `Image` | 앨범 | AlbumPage |
| `my` | `User` | 내정보 | MyPage |

#### 스타일

- **배경**: 반투명 `bg-white/80 backdrop-blur-xl` (다크모드: `bg-gray-900/80`)
- **테두리**: 상단 `border-t border-gray-200`
- **높이**: `h-16` (64px)
- **고정 위치**: `fixed bottom-0 left-0 right-0 z-50`
- **Safe Area**: `pb-safe` (iOS 홈 인디케이터 대응)

#### 활성 상태 표시

- **아이콘**: `text-blue-600` (활성) / `text-gray-500` (비활성)
- **라벨**: `font-medium` (활성) / 일반 (비활성)
- **애니메이션**: 활성 탭 아래 파란 점 (`bg-blue-600`, 4x4px) 표시
- **호버 효과**: 아이콘 스케일 업 (1.1x), 위로 2px 이동

#### Props

```typescript
interface BottomNavProps {
  activeTab: 'home' | 'schedule' | 'boards' | 'album' | 'my';
  onTabChange: (tab: 'home' | 'schedule' | 'boards' | 'album' | 'my') => void;
}
```

#### 주요 섹션

##### 1. Hero Stats (통계 카드)

**위치**: 상단
**레이아웃**: `grid grid-cols-3 gap-3`

| 카드 | 배경색 | 아이콘 | 숫자 | 라벨 |
|------|--------|--------|------|------|
| 활동 멤버 | `from-blue-500 to-blue-600` | `Users` | 24 | 활동 멤버 |
| 예정 일정 | `from-green-500 to-green-600` | `Calendar` | 3 | 예정 일정 |
| 최근 경기 | `from-purple-500 to-purple-600` | `Trophy` | 8-5 | 최근 경기 |

**스타일**: 카드 형태, 흰색 텍스트, 그라데이션 배경

##### 2. Deadline Alert (마감 임박 알림)

**위치**: Hero Stats 아래
**표시 조건**: 다음 일정이 있고, 마감까지 48시간 이내일 때

```
┌─────────────────────────────────────────┐
│ ⏰ 출석 투표 마감 임박!                 │
│ 경기 vs OO팀                            │
│ 12시간 후 마감 (전날 23:00)             │
└─────────────────────────────────────────┘
```

**스타일**: `from-orange-500 to-red-500` 그라데이션, 흰색 텍스트

##### 3. 다가오는 일정 섹션

**제목**: "📅 다가오는 일정" + "전체보기" 버튼 (우측 정렬)

**아이템 구성** (최대 3개):
- 날짜 (`format(post.startAt, 'yyyy.MM.dd')`)
- 제목 (`post.title`)
- 시간 및 장소 (`⏰ ${format(post.startAt, 'HH:mm')} @ ${post.place}`)
- 참석 인원 (`👥 참석 ${post.attendanceSummary?.attending || 0}명`)

**클릭 동작**: `onNavigate('schedule')`

##### 4. 최신 공지 섹션

**제목**: "📢 최신 공지" + "전체보기" 버튼 (우측 정렬)

**아이템 구성** (최대 2개):
- 핀 아이콘 (`📌`) - `post.pinned === true`일 때만
- 타입 뱃지 (`[공지]`)
- 제목 (`post.title`)
- 작성자 및 시간 (`post.author.name | ${formatDistanceToNow(post.createdAt)}`)
- 댓글 개수 (`💬 ${post.commentCount || 0}개 댓글`)

**클릭 동작**: `onNavigate('boards')`

#### Props

```typescript
interface HomePageProps {
  onNavigate: (tab: 'schedule' | 'boards' | 'album', postId?: string) => void;
}
```

## 7. 반응형 및 다크모드

### 7.1 반응형 브레이크포인트

- **모바일 우선**: 기본 스타일은 모바일 기준
- **최대 너비**: `max-w-md` (약 448px) - 데스크톱에서도 모바일 레이아웃 유지
- **그리드**: 모바일에서는 1열, 카드 형태는 3열 그리드 사용

### 7.2 다크모드

- **테마 토글**: SettingsPage에서 전환 가능
- **적용 범위**: 모든 컴포넌트에 `dark:` 접두사로 다크모드 스타일 정의
- **색상 체계**:
  - 배경: `bg-gray-50` (라이트) / `bg-gray-950` (다크)
  - 텍스트: `text-gray-900` (라이트) / `text-white` (다크)
  - 카드: `bg-white` (라이트) / `bg-gray-900` (다크)

## 8. 애니메이션

### 8.1 페이지 전환

- **Framer Motion** 사용
- **초기 애니메이션**: `initial={{ opacity: 0, y: 20 }}`
- **진입 애니메이션**: `animate={{ opacity: 1, y: 0 }}`
- **지연 시간**: 각 섹션별 `transition={{ delay: 0.1 * index }}`

### 8.2 인터랙션 피드백

- **버튼 클릭**: `whileTap={{ scale: 0.95 }}`
- **호버 효과**: `hover:bg-*` 클래스 사용
- **탭 전환**: 활성 탭 아래 파란 점 애니메이션 (`layoutId="activeTab"`)
